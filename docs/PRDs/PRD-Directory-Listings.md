# PRD: Directory Listings

## Problem

The map and discovery experience feels sparse while the vendor network
grows.

## Goal

Allow users to discover local businesses that are not yet RootSync
vendors.

## User Stories

-   As a visitor, I want to see local businesses near me.
-   As a directory business, I may later claim my listing.
-   As a vendor, I want verified vendors to stand out.

## Functional Requirements

-   View-only directory listings
-   Distinct "Directory Listing" badge
-   Search and map integration
-   Filters for Vendors vs Directory Listings
-   Future "Claim this listing" workflow

## Acceptance Criteria

-   Listings appear on map/search.
-   Users cannot message or purchase from directory listings.
